# 取扱説明書を読むゲーム

説明書が更新されるたびにルールを選び、横スクロールの遊び方を育てていくMVPです。  
Vite + Vue + TypeScriptで、ルール定義・ゲームロジック・UIを分けています。

## 起動

```bash
npm install
npm run dev
```

本番ビルド確認:

```bash
npm run build
```

## 実装の分割

- `src/data/manualDeck.ts`: 説明書バージョン、2択、操作キー、危険色、派生パラメータ。
- `src/data/gameBalance.ts`: 更新距離、ゴール距離、スコア比率、投擲スコア重み。
- `src/domain/types.ts`: ルール、スコア、ゲーム状態の型。
- `src/domain/ruleEngine.ts`: 説明書データと選択履歴から実行時ルールを合成。
- `src/game/sideScroller.ts`: Canvas横スクロール本体。
- `src/game/throwEngine.ts`: 説明書投擲フェーズと投擲スコア。
- `src/composables`: Vueとゲームエンジンをつなぐ状態管理。
- `src/components`: HUD、説明書、選択UI、投擲UI。

## MVPで入っているもの

- 説明書の段階更新とジャンルカード選択。
- 危険色の切替。
- ジャンプキー変更。
- 音ゲー: 拍による危険色反転、ジャスト入力ボーナス、拍ダッシュ。
- STG: 前方ショット、3WAYショット、連射テンポ、撃破コンボ。
- タワーディフェンス: タワー設置、自動射撃、防衛ライン、予備タワー。
- 空中/走行: 二段ジャンプ、長めの滞空、オートラン、低速精密。
- 複合: 拍撃ちショット、砲台弾幕、拍動タワー、空中ショットライン。
- 任意タイミングの説明書投擲。
- 横スクロール70% + 投擲30%の最終スコア。

## 次に拡張しやすい場所

ルールを増やす場合は、まず `src/data/manualDeck.ts` にジャンルカードを追加します。  
`links` は「そのカードが形作るジャンル要素」、`requiresGenres` は複合カードの解放条件です。
